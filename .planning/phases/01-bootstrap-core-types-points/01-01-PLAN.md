    To jest checkpoint:human-verify — Claude NIE wykonuje instalacji autonomicznie. Użytkownik
    uruchamia komendy ręcznie (instalacja systemu wymaga interakcji shell-level i potencjalnie
    zmiany PATH w `~/.zshrc`). Claude dostarcza dokładne komendy i czeka na "approved" / opis błędu.

    Komendy do uruchomienia ręcznie przez użytkownika (skopiuj-wklej):

    ```bash
    # 1. Instalacja juliaup (curl jest preferowany — instalator official; --yes pomija interaktywne pytania)
    curl -fsSL https://install.julialang.org | sh -s -- --yes

    # 2. Aktywacja zmian PATH w bieżącej sesji shellowej (juliaup dodaje wpis do ~/.zshrc)
    source ~/.zshrc

    # 3. Weryfikacja — powinno wypisać 1.10.x lub nowszą
    julia --version

    # 4. (Opcjonalnie) Przypnij default channel do 1.10 LTS dla broad-compat reproducibility:
    juliaup add 1.10
    juliaup default 1.10
    ```

    Alternatywa Homebrew (NIE zalecana — juliaup ma lepsze multi-version support):
    `brew install juliaup` → `juliaup add 1.10` → `juliaup default 1.10`.

    Po wykonaniu instalacji przez użytkownika — Claude weryfikuje przez automated check (sekcja
    `<verify>` poniżej).
  </action>
  <what-built>

    (obecnie 1.12, lub `1.11` jeśli użytkownik preferuje stabilność). Po instalacji `julia --version`

    3. Uruchom: `julia -e 'println(VERSION)'`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → user laptop (curl install) | Skrypt `install.julialang.org` jest pobierany przez HTTPS z oficjalnej domeny julialang.org |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | curl install script (supply chain) | mitigate | Użyj oficjalnego URL `install.julialang.org` (DNSSEC + HTTPS); nie kopiuj instalatora z third-party. Alternatywnie pobierz tarball z [julialang.org/downloads](https://julialang.org/downloads/) z weryfikacją SHA256. |
| T-01-02 | Elevation of Privilege | juliaup PATH injection | accept | juliaup instaluje pod `$HOME/.juliaup` (user-space, bez sudo). Dodaje wpis do `~/.zshrc` — standardowa praktyka dev-tooli (rustup, pyenv, nvm). Niskie ryzyko. |
| T-01-03 | Denial of Service | Brak Julii blokuje całą fazę | mitigate (już zaimplementowane przez ten plan) | Plan 01 jest pierwszy; każdy kolejny plan w tej fazie zakłada `julia` w PATH. Acceptance criterion plana 01 weryfikuje `julia --version` przed unblocked'em wave 2. |
</threat_model>
